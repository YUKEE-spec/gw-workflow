# Examples（省部级场景）

## Example 1：汇报材料 — 对话澄清 → 确认单（不写正文）

**Input：**

```text
帮我写一份向省政府常务会汇报的数字政府建设材料。
```

**Expected Agent behavior（先问，不直接写全文）：**

```text
先帮您把任务对齐，方便后面写扎实。请确认或补充：

1. 汇报主体是哪个厅局？预计书面汇报还是会上口头汇报？
2. 是否有背景材料可 @：上级部署文件、专项方案、统计数据、往期汇报版本？
3. 希望突出哪 1–2 条（如：数据共享、基层减负、安全可控）？篇幅大约多少字？
4. 除默认禁用词外，本稿有无特别不能写的表述？

您可先贴 3–5 条要点或一份背景材料，我再出《任务确认单》。
```

## Example 2：省政府常务会汇报 — 正文片段（省级）

**Input（任务包已齐）：**

```text
省数据局向省政府常务会书面汇报。依据：国务院关于加强数字政府建设的指导意见及我省实施方案。
要点：① 省一体化政务服务平台接入部门〔N〕个，高频事项网办率〔X〕%；② 建立跨部门数据共享协调机制，累计挂接目录〔M〕条；
③ 问题：部分垂管系统仍不通，基层重复填报；④ 建议：省级统筹未贯通系统清单，〔季度〕前完成〔数量〕项对接。
约 2500 字，要问题与建议，不要夹请示。
```

```text
关于我省数字政府建设推进情况的汇报

一、主要进展

（一）平台支撑能力持续增强。省一体化政务服务平台累计接入省直部门〔N〕个，国家层面明确的省级高频政务服务事项网办率达到〔X〕%。〔部门名称〕牵头建立跨部门数据共享协调机制，省级数据目录累计挂接〔M〕条，〔具体业务名称〕等〔数量〕个场景实现“数据多跑路、基层少填报”。

（二）……

二、存在问题

一是垂管系统与省级平台仍未完全贯通，部分事项仍需基层在多个系统重复录入。二是……

三、下一步建议

省级层面建立未贯通系统清单，实行台账管理，明确牵头单位、完成时限和验收标准，力争〔日期〕前完成首批〔数量〕项系统对接。同时……

省数据局
〔日期〕
```

## Example 3：部党组会讲话稿 — 问诊要点

**Input：**

```text
起草部长在部党组扩大会上的讲话，部署年度行业监管重点任务。
```

**Agent 应优先问：**

- 讲话时长 / 字数？是否已有部长定调表述或必须引用的文件？
- 听众范围（党组成员、司局、直属单位）？动员型还是总结部署型？
- 背景材料：年度工作会议文件、统计公报、典型问题案例？

## Example 4：润色 + 敏感词替换

**Input：**

```text
请降AI味并检查敏感词：
为深化放管服改革，推进一件事一次办，我厅持续优化营商环境，推动政务服务提质增效，取得显著成效。
```

**Expected：**

```text
【正文】
我厅按照转变政府职能、优化政务服务的部署，梳理高频事项材料清单，将〔事项类型〕申请材料由〔N〕项压减至〔M〕项，在〔平台名称〕上线集成办理模块。今年以来，〔具体指标〕〔变化情况，如有数据则写〕。

【修改说明】
- 敏感词：「放管服改革」→「转变政府职能、优化政务服务」；「一件事一次办」→「集成办理模块」（避免品牌连用）
- 降AI味：删除无宾语「推动…提质增效、取得显著成效」，改为具体举措与指标占位
- 待补充：事项类型、材料项数、平台名、指标数据
```

## Example 5：调研报告 — 任务确认单

**Input：**

```text
写一份关于中西部省份承接产业转移的调研报告，给部领导参考。
```

**Expected《任务确认单》骨架：**

```text
【任务确认单】
层级/主体：〔部委名称〕调研组
文种/场景：调研报告 → 部领导决策参考
已有输入：（待用户 @ 调研方案、座谈记录、数据表）
建议结构：背景→承接现状→问题→原因→建议（问题建议对应）
篇幅与形式：约 5000 字，可不套红头
重点突出：（待用户确认 1–2 条）
禁用/敏感：默认禁用表 + （用户补充）
待您确认或补充：调研时间、样本省份、是否涉密
```
