# 敏感词、错词替换与引用纪律

## 错词词库（主表）

**权威来源**：[`错词词库-20260708.csv`](错词词库-20260708.csv)（约 180+ 条，用户原用词库）。

起草、润色、定稿前**必须加载并检索**该 CSV。用户更新词库时，替换同名文件或追加词条（保留首行表头）。

### 列含义

| 列 | 字段 | 说明 |
| --- | --- | --- |
| 1 | 错误词 | 正文中不得保留的表述；含通配符时按规则匹配 |
| 2 | 建议词 | 替换用词；若为处置说明（见下），按说明处理而非机械替换 |
| 3 | 备注 | 可选；CSV 首行说明行可忽略 |

### 通配符

| 符号 | 含义 | 示例 |
| --- | --- | --- |
| `*` | 任意字符（0 个或多个） | `本地*优先` → 匹配「本地产品优先」 |
| `?` | 单个字符 | `党的?大` → 匹配「党的二十大」 |
| `**` | 按 CSV 原义，通常表示可变片段 | `支持**代表性企业` → 整句审查 |

匹配时：**宁可多查、不可漏查**；不确定则向用户确认。

### 建议词两类（勿一律机械替换）

**A. 可直接替换**（第 2 列为目标表述）

例：`一件事一次办` → `高效办成一件事`；`截止` → `截至`；`中共` → `中国共产党`。

**B. 处置说明**（第 2 列为原则，须理解后改写）

| 错词示例 | 建议词（说明） | 处理 |
| --- | --- | --- |
| 万众创新 | 过时的提法，优化 | 删除或改为现行表述，勿照搬「万众创新」 |
| 工作专班 | 谨慎使用… | 无上级批复依据时不写；有则写规范全称 |
| 支持**代表性企业做大做强 | 涉嫌违背公平竞争审查条例 | 删除或改为公平竞争表述 |
| 同考核 | 与基层减负精神不符 | 删除层层考核、重复考核类表述 |
| 市场主体 / 放管服 | 除已有文件外，必须修改 | 非引用原文时一律改写 |
| 报送贯彻落实情况 等 | 直接提出具体落实措施 | 少写「传达学习」，多写可执行举措 |
| 一票否决 / 责任状 等 | 建立科学考核评价体系 | 删形式主义考核表述 |
| *台账 / *记录 等 | 注重工作实效 | 减留痕、减台账，写实质工作 |

## 绝对禁用（与 CSV 一致，摘要）

全文**不得出现**（含变体、固定搭配），定稿前必检：

| 禁用表述 | 建议替换（见 CSV） |
| --- | --- |
| **放管服改革** / **放管服**（非文件原文引用） | 转变政府职能、优化政务服务等 |
| **一件事一次办** / **最多跑一次** | **高效办成一件事** |
| **市场主体**（非文件原文引用） | 按语境改为经营主体、企业等 |

CSV 中其余词条：**命中即改**；《修改说明》中注明，**正文不出现修改说明**。

## 用户自定义

- 当次对话补充的禁用词，与 CSV **同等强制**。
- 用户可在 Skill 使用中追加；Agent 记入当次任务并全文检索。

## 执行规则

1. **起草前**：加载 `错词词库-20260708.csv`，知悉通配符与 B 类说明词。
2. **润色中**：主动替换用户材料中的错词；文件原文引用可保留，但须标注「原文引用」并在《任务确认单》说明。
3. **定稿前**：按 CSV 全文检索（含通配模式）；敏感词合规项未通过则**不得交付**（见 [reference-style.md](reference-style.md) 评分）。
4. 不确定时**宁可稳妥改写**并向用户确认。

## 政策与事实引用纪律

### 文件与会议

- 引用中央、国务院、部委、省委省政府文件：用**规范全称或简称**；无材料依据不写文号、不写「精神」类概括。
- 未提供原文时，不编造「根据××文件精神」；用 `〔文件名称及文号〕` 占位或向用户索取。

### 领导职务与姓名

- **不编造**领导姓名、职务、排序。
- 无材料时：`〔职务+姓名〕` 或「有关领导」等稳妥表述，并列入待补充。

### 数据与统计

- 数据须来自用户材料或权威口径说明；不臆造增幅、排名、比例。
- 涉及「全国/全省/全行业」对比时，须有来源；否则改为「据有关部门统计」或占位。
- 标注统计时间范围（如「2025年1—11月」）。

### 政策表述

- 不自行创造新提法；与现行公开表述不一致时，以用户提供的背景材料为准。
- 涉及改革、营商环境、政务服务等，遵守本 Skill 错词词库及用户补充禁忌。
