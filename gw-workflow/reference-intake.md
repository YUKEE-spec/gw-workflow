# 起草前对话澄清（详细）

## 何时必须走对话澄清

| 任务类型 | 是否对话澄清 |
| --- | --- |
| **从零起草**（汇报材料、讲话稿、调研报告、方案、总结等） | **必须**，除非用户已提供完整任务包 |
| **润色 / 降AI味 / 格式修正**（已有全文） | 简短确认修改重点后直接处理 |
| **局部扩写 / 压缩 / 改写某段** | 确认段落目标即可 |
| 用户明确「先按现有信息写一版」 | 可跳过，但须标注 `〔待补充〕` |

**完整任务包**（满足则可少问）：文种 + 场合 + 对象 + 层级（省/部）+ 核心要点（≥3条具体事实，含政策依据或数据口径之一）+ 篇幅 + 至少一类输入材料。

## 三阶段问诊

每轮 **3–5 问**；根据回答追问。确认后输出《任务确认单》，用户确认再起草。

### 阶段 A：任务定位

- **层级与主体**：省级还是部级？以哪个机关/部门名义成稿？
- **文种**：汇报材料 / 讲话稿 / 调研报告 / 方案 / 通知 / 请示 / 专报 / …
- **场景**：什么会议/场合/书面或口头？
- **对象**：哪位领导、哪个会议、上级机关还是内部？
- **目的**：通报情况 / 争取支持 / 提请决策 / 经验推介 / 回应关切？

### 阶段 B：输入收集（必问背景材料）

> 是否有可参考或必须依据的**背景材料**？上级文件、领导批示、专项方案、统计数据、调研记录、兄弟单位材料、往期汇报、会议纪要、PPT 要点等。请 @ 文件、粘贴要点或说明路径。

| 输入类型 | 说明 |
| --- | --- |
| 背景材料 | 政策依据、上级部署、历史沿革 |
| 事实材料 | 数据、案例、项目进展、时间节点 |
| 参考范文 | 领导认可的往期写法（学结构不照搬） |
| 口述/散点 | 领导口头要求、会议共识 |
| 约束材料 | 必写内容、必避内容、敏感词补充 |

- 一句话任务 → **不立刻写全文**，先追问输入。
- 有附件 → **先阅读**再针对空白提问。
- 不得虚构；缺失用 `〔待补充〕`。

### 阶段 C：输出规格

篇幅、结构、重点（1–2条）、语气站位、形式要素、禁忌、时限、是否先出提纲。

## 《任务确认单》模板

```text
【任务确认单】
层级/主体：
文种/场景：
主体 → 对象：
核心目的：
已有输入：（背景材料/数据/要点/范文）
建议结构：
篇幅与形式：
重点突出：
禁用/敏感：
待您确认或补充：
```

## 文种专属问诊

### 汇报材料

1. 场合：常委会 / 常务会 / 办公会 / 专题会 / 口头 / 书面？
2. 对象与决策需求：仅通报还是需建议/协调？（与请示区分）
3. 背景材料：部署文件、批示、方案、统计口径、往期版本、兄弟参考？
4. 核心事实：进展、数据、项目、问题？
5. 输出：字数/时长、体例、问题深度、建议是否要可操作建议？

**默认结构**：背景与依据 → 总体情况 → 主要做法与进展 → 成效与亮点 → 问题与原因（简要）→ 下一步或建议。

### 讲话稿 / 发言稿

1. 会议类型与性质：部署会 / 总结会 / 座谈 / 仪式 / 电视电话会？
2. **时长与字数**（口头必问）
3. 听众：层级、人数、是否含主要领导、是否需要兼顾多方？
4. 是否已有**领导定点**或必须体现的表述？能否引用讲话精神原文？
5. 背景材料：会议方案、议程、前期文件、典型材料？
6. 基调：动员型 / 总结型 / 研究型 / 表态型？收束要不要提具体任务？

**默认结构**：称谓 → 开场（依据/形势）→ 认识判断 → 重点任务 → 落实要求 → 收束。

### 调研报告

1. 调研主题与**授权依据**（谁部署、什么范围）
2. 调研方式：实地 / 座谈 / 问卷 / 数据比对？样本与时间？
3. 是否涉密或**不宜公开**的内容边界？
4. 背景材料：调研方案、提纲、原始记录、统计表？
5. 建议性质：供决策参考 / 供部门落实 / 供宣传？是否要「谁来做、何时完成」？
6. 篇幅与是否需「问题—建议」一一对应？

**默认结构**：调研背景 → 基本情况 → 主要问题 → 原因分析 → 对策建议。

### 工作方案

1. 对应哪一项**上级部署或本级决策**？
2. 实施周期与**关键节点**（考核、验收、报送）？
3. 背景材料：上级方案、现状摸底、试点经验？
4. 责任分工口径：牵头/配合/属地/行业条线如何表述？
5. 是否需要附件（任务清单、指标体系、路线图）？
6. 与「计划」「通知」的边界：本文是方案正文还是随通知下发？

**默认结构**：指导思想/目标 → 重点任务 → 步骤安排 → 责任分工 → 保障措施。

## 润色/改稿输出格式

交付时分两块（**正文不含修改说明**）：

```text
【正文】
（可交付稿）

【修改说明】
- 敏感词：原表述 → 改写（如有）
- 站位/结构：…
- 仍待补充：…
```

## 分轮提问检查清单

**第一轮**：文种、场景、主体、对象、目的  
**第二轮**：背景材料、核心要点、参考范文  
**第三轮**：篇幅、结构、重点、敏感词、是否先出提纲

用户一次性给全时，整理为《任务确认单》即可，勿机械三轮。
